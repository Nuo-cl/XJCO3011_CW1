# 项目基本陈述：ScholarTrack — arXiv 论文追踪与研究笔记平台

## 项目概述

ScholarTrack 是一个面向研究人员和学生的 RESTful Web API，围绕 arXiv 论文构建「发现 → 收藏 → 笔记 → 复习」的研究工作流。用户可以追踪特定领域（如计算机视觉、LLM）的最新论文动态，将感兴趣的论文收藏并分类管理，撰写阅读笔记，并通过间隔重复机制巩固关键知识。系统集成 RAG 语义搜索，支持用户以自然语言在论文库和个人笔记中检索信息。

## 核心理念

- **以用户的研究活动为核心**：论文是素材，不是主体。API 管理的是用户的研究工作流，而非论文内容本身。
- **互动而非查询**：用户在平台上持续「做事」——收藏、写笔记、复习、搜索，而不是被动浏览数据。
- **单一数据来源**：所有论文均来自 arXiv API，不涉及用户上传的本地文档，保持数据源清晰。
- **API 即产品，MCP 是主要交互方式**：不提供前端页面或终端 UI。API 通过 Swagger UI 供开发者/评审测试，通过 MCP 供 AI 助手驱动完整工作流（搜索论文 → 收藏 → 写笔记 → 复习）。

## 主要功能模块

### 1. 用户系统
- 注册、登录（JWT 认证）
- 用户个人信息与偏好设置（关注的研究领域）

### 2. 论文发现
- 从 arXiv API 获取特定领域的最新论文
- 按关键词、分类（cs.CV、cs.CL 等）、日期范围搜索论文
- 论文元数据本地缓存（标题、作者、摘要、链接），避免重复请求

### 3. 论文管理
- 收藏论文到个人库
- 为收藏的论文添加标签和分类
- 公共论文池去重：同一篇论文只存储一次，多用户通过收藏关系关联

### 4. 笔记系统
- 为收藏的论文撰写阅读笔记（完整 CRUD）
- 笔记支持 Markdown 格式
- 笔记可独立存在，也可关联到特定论文

### 5. 闪卡与间隔重复
- 基于笔记内容生成复习闪卡（问答对）
- SM-2 间隔重复算法安排复习时间
- 复习记录追踪（正确率、复习次数、下次复习时间）

### 6. RAG 语义搜索
- 论文摘要和用户笔记向量化存入 ChromaDB
- 语义搜索论文：「关于 Transformer 注意力机制优化的论文」
- 语义搜索笔记：「我之前读过的关于数据增强的笔记」
- 跨库联合搜索：同时检索论文摘要和个人笔记

### 7. MCP Server
- 暴露结构化工具供 AI 助手调用
- 工具示例：搜索论文、收藏论文、创建笔记、查询知识库、开始复习

## 技术架构

```
用户 / AI 助手 (MCP)
        ↓
   Flask REST API（JWT 认证）
        ↓
  ┌─────────┬──────────────┐
  │ SQLite  │  ChromaDB    │
  │ (主存储) │ (向量搜索)    │
  └────┬────┴──────────────┘
       │
   arXiv API（外部数据源）
```

- **SQLite + SQLAlchemy**：用户、论文、收藏、笔记、闪卡等所有核心数据的 CRUD
- **ChromaDB**：论文摘要和笔记内容的向量化索引，提供语义搜索能力
- **arXiv API**：论文数据的唯一外部来源
- **Flask + JWT**：RESTful API 框架与认证
- **pytest**：自动化测试
- **PythonAnywhere**：可选部署平台（优先保证本地运行 + MCP 演示）

## 项目范围边界

### 包含
- 论文元数据的获取、缓存与管理
- 用户笔记的完整 CRUD
- JWT 认证与受保护端点
- RAG 语义搜索
- 间隔重复复习系统
- MCP Server 接口
- API 文档与技术报告

### 不包含
- 论文 PDF 原文的存储或渲染
- 用户上传本地文档
- 论文推荐算法（基于协同过滤等）
- 社交功能（关注、分享、评论他人笔记）
- 前端界面或终端 UI（纯 API 项目，通过 Swagger UI 测试，通过 MCP 演示工作流）

## 演示策略

口试时采用双层演示：
1. **Swagger UI** — 快速展示 REST API 端点、请求/响应格式、状态码（证明 REST 基础扎实）
2. **MCP + AI 对话** — 展示完整研究工作流：AI 助手搜索论文、收藏、写笔记、语义搜索知识库、复习闪卡（展示高级特性与 GenAI 集成）
