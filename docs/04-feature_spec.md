|------|------|--------|------|

## F1 用户认证

### 功能描述

提供用户注册与登录功能，通过 JWT token 保护需要身份验证的端点。

### 详细需求

#### F1.1 用户注册
- 用户提交用户名、邮箱、密码进行注册
- 用户名和邮箱必须唯一
- 密码以哈希形式存储（bcrypt/werkzeug）
- 注册成功返回用户信息（不含密码）

#### F1.2 用户登录
- 用户提交用户名（或邮箱）+ 密码登录
- 验证成功返回 JWT access token
- Token 包含用户 ID，设置合理过期时间

#### F1.3 认证保护
- 需要认证的端点通过 `Authorization: Bearer <token>` 验证
- 未认证请求返回 401
- Token 过期返回 401 并提示刷新

#### F1.4 用户信息
- 用户可查看自己的个人信息
- 用户可更新偏好设置（关注的研究领域列表）

### 端点概览

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/auth/register` | 否 | 注册 |
| POST | `/api/auth/login` | 否 | 登录 |
| GET | `/api/users/me` | 是 | 获取个人信息 |
| PUT | `/api/users/me` | 是 | 更新个人信息/偏好 |

## F2 论文发现

### 功能描述

通过 arXiv API 获取论文数据，支持按领域、关键词、日期搜索，并将结果缓存到本地数据库。

### 详细需求

#### F2.1 论文搜索
- 支持按关键词搜索 arXiv 论文
- 支持按分类过滤（cs.CV、cs.CL、cs.AI、cs.LG 等）
- 支持按日期范围过滤
- 支持分页（page、per_page 参数）
- 返回论文列表：标题、作者、摘要、分类、发布日期、arXiv 链接、PDF 链接

#### F2.2 论文详情
- 通过 arXiv ID 获取单篇论文详情
- 如果本地数据库已缓存则直接返回，否则从 arXiv API 获取并缓存

#### F2.3 热门论文
- 获取指定领域近期论文（按日期排序）
- 默认返回最近 7 天的论文

#### F2.4 本地缓存策略
- 从 arXiv 获取的论文元数据自动存入 SQLite（公共论文池）
- 通过 arxiv_id 去重，已存在的论文不重复插入
- 缓存的论文摘要同步写入 ChromaDB 用于语义搜索

### 端点概览

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| GET | `/api/papers/search` | 否 | 搜索 arXiv 论文 |
| GET | `/api/papers/trending` | 否 | 获取指定领域近期热门论文 |
| GET | `/api/papers/<arxiv_id>` | 否 | 获取单篇论文详情 |

## F3 论文管理

### 功能描述

用户可以收藏论文到个人库，并通过标签进行分类管理。

### 详细需求

#### F3.1 收藏论文
- 用户可将论文添加到个人收藏
- 收藏时可附带个人备注
- 重复收藏同一论文返回提示（409 Conflict）

#### F3.2 取消收藏
- 用户可移除收藏的论文
- 移除收藏不影响公共论文池中的数据

#### F3.3 查看收藏
- 获取当前用户的所有收藏论文
- 支持分页
- 支持按标签过滤

#### F3.4 标签管理
- 用户可为收藏的论文添加标签（如"attention mechanism"、"to read"、"important"）
- 用户可查看自己所有标签
- 用户可按标签筛选收藏的论文
- 用户可移除论文上的标签

### 端点概览

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/papers/<arxiv_id>/save` | 是 | 收藏论文 |
| DELETE | `/api/papers/<arxiv_id>/save` | 是 | 取消收藏 |
| GET | `/api/library` | 是 | 查看个人收藏列表 |
| POST | `/api/library/<arxiv_id>/tags` | 是 | 为收藏论文添加标签 |
| DELETE | `/api/library/<arxiv_id>/tags/<tag>` | 是 | 移除标签 |
| GET | `/api/tags` | 是 | 查看用户所有标签 |

## F4 笔记系统

### 功能描述

用户可以为收藏的论文撰写阅读笔记，支持完整的 CRUD 操作。这是项目的核心交互功能，也是满足作业 CRUD 要求的主要模型。

### 详细需求

#### F4.1 创建笔记
- 用户可为收藏的论文创建笔记
- 笔记包含标题和内容（支持 Markdown）
- 一篇论文可以有多条笔记
- 创建笔记时，内容同步向量化写入 ChromaDB

#### F4.2 查看笔记
- 查看指定论文的所有笔记
- 查看当前用户的所有笔记（跨论文）
- 支持分页

#### F4.3 更新笔记
- 用户可编辑自己的笔记标题和内容
- 更新后同步更新 ChromaDB 中的向量
- 只能编辑自己的笔记（403）

#### F4.4 删除笔记
- 用户可删除自己的笔记
- 删除后同步移除 ChromaDB 中的向量
- 只能删除自己的笔记（403）

### 端点概览

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/notes` | 是 | 创建笔记 |
| GET | `/api/notes` | 是 | 查看用户所有笔记 |
| GET | `/api/papers/<arxiv_id>/notes` | 是 | 查看指定论文的笔记 |
| GET | `/api/notes/<id>` | 是 | 查看单条笔记 |
| PUT | `/api/notes/<id>` | 是 | 更新笔记 |
| DELETE | `/api/notes/<id>` | 是 | 删除笔记 |

## F5 闪卡与间隔重复

### 功能描述

用户可以基于笔记创建复习闪卡，系统通过 SM-2 间隔重复算法安排复习计划，帮助用户巩固论文中的关键知识。

### 详细需求

#### F5.1 创建闪卡
- 用户可手动创建闪卡（问题 + 答案）
- 闪卡关联到具体笔记
- 新闪卡初始参数：ease_factor=2.5, interval=1, next_review=创建时间

#### F5.2 查看闪卡
- 查看某条笔记下的所有闪卡
- 查看当前用户所有闪卡
- 获取今天到期需要复习的闪卡

#### F5.3 更新与删除闪卡
- 用户可编辑闪卡的问题和答案
- 用户可删除闪卡

#### F5.4 复习流程
- 用户获取待复习的闪卡
- 用户提交复习结果（评分 0-5，SM-2 标准评分）
- 系统根据 SM-2 算法更新：ease_factor、interval、next_review_at
- 记录每次复习的结果（正确/错误、时间）

#### F5.5 复习统计
- 查看用户的复习统计：总卡片数、今日待复习、已掌握、复习历史趋势

### SM-2 算法简述

```
如果评分 >= 3（记住了）:
    interval = 上次 interval × ease_factor
    ease_factor = ease_factor + (0.1 - (5 - 评分) × (0.08 + (5 - 评分) × 0.02))
如果评分 < 3（忘了）:
    interval = 1（重新开始）
ease_factor 最小值 = 1.3
next_review_at = 当前时间 + interval 天
```

### 端点概览

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/flashcards` | 是 | 创建闪卡 |
| GET | `/api/flashcards` | 是 | 查看用户所有闪卡 |
| GET | `/api/notes/<id>/flashcards` | 是 | 查看某笔记的闪卡 |
| PUT | `/api/flashcards/<id>` | 是 | 编辑闪卡 |
| DELETE | `/api/flashcards/<id>` | 是 | 删除闪卡 |
| GET | `/api/flashcards/due` | 是 | 获取今日待复习闪卡 |
| POST | `/api/flashcards/<id>/review` | 是 | 提交复习结果 |
| GET | `/api/review/stats` | 是 | 复习统计 |

## F6 RAG 语义搜索

### 功能描述

利用 ChromaDB 对论文摘要和用户笔记进行向量化存储，提供自然语言语义搜索能力。

### 详细需求

#### F6.1 论文语义搜索
- 用户输入自然语言查询（如"关于 Transformer 注意力机制优化的论文"）
- 在论文摘要向量库中检索最相关的论文
- 返回论文列表，按相关度排序

#### F6.2 笔记语义搜索
- 用户搜索自己的笔记（如"我之前写的关于数据增强的笔记"）
- 只检索当前用户的笔记（按 user_id 过滤）
- 返回笔记列表，按相关度排序

#### F6.3 跨库联合搜索
- 同时检索论文摘要和用户笔记
- 合并结果并标注来源类型（paper/note）
- 按相关度统一排序

#### F6.4 向量同步
- 论文缓存到本地时，摘要自动向量化写入 ChromaDB
- 笔记创建/更新/删除时，同步维护 ChromaDB 中对应向量
- 使用 sentence-transformers 或 ChromaDB 内置 embedding

### 端点概览

| 方法 | 路径 | 认证 | 说明 |
|------|------|------|------|
| POST | `/api/search/papers` | 否 | 论文语义搜索 |
| POST | `/api/search/notes` | 是 | 笔记语义搜索 |
| POST | `/api/search/all` | 是 | 跨库联合搜索 |

## F7 MCP Server

### 功能描述

将 API 的核心功能封装为 MCP 工具，供 AI 助手（如 Claude）调用，实现完整的研究工作流自动化。

### 详细需求

#### F7.1 工具定义

| 工具名 | 功能 | 对应 API |
|--------|------|----------|
| `search_papers` | 搜索 arXiv 论文 | F2.1 |
| `get_trending_papers` | 获取领域热门论文 | F2.3 |
| `save_paper` | 收藏论文 | F3.1 |
| `list_library` | 查看收藏列表 | F3.3 |
| `create_note` | 为论文创建笔记 | F4.1 |
| `search_notes` | 语义搜索笔记 | F6.2 |
| `search_knowledge` | 跨库联合搜索 | F6.3 |
| `get_due_flashcards` | 获取待复习闪卡 | F5.4 |
| `review_flashcard` | 提交复习结果 | F5.4 |

#### F7.2 实现方式
- 使用 `mcp` Python SDK
- MCP server 作为独立入口文件，内部复用 Flask app 的业务逻辑层
- 不经过 HTTP 层，直接调用 service/数据库层函数

#### F7.3 认证处理
- MCP 工具运行在本地，默认使用预配置的用户身份
- 或通过 MCP server 启动参数指定用户

### 演示工作流示例

```
用户: "帮我搜一下最近关于 Vision Transformer 的论文"
AI → search_papers(query="Vision Transformer", category="cs.CV", days=7)
AI: "找到 15 篇相关论文，以下是最相关的 5 篇..."

用户: "收藏第二篇，写一条笔记：这篇提出了新的多尺度注意力机制"
AI → save_paper(arxiv_id="2403.xxxxx")
AI → create_note(arxiv_id="2403.xxxxx", title="多尺度注意力", content="...")

用户: "我之前看过哪些关于注意力机制的论文？"
AI → search_knowledge(query="注意力机制")
AI: "找到 3 篇收藏的论文和 2 条笔记..."

用户: "今天有需要复习的闪卡吗？"
AI → get_due_flashcards()
AI: "有 5 张闪卡待复习，第一张：什么是 Multi-Head Attention？"
```

## 全局需求

### G1 错误处理
- 所有端点使用统一的错误响应格式：`{"error": "<错误类型>", "message": "<描述>"}`
- 正确使用 HTTP 状态码：200、201、400、401、403、404、409、500
- 不暴露内部堆栈信息

### G2 分页
- 列表端点统一支持分页参数：`page`（默认 1）、`per_page`（默认 20，最大 100）
- 响应包含分页元数据：`total`、`page`、`per_page`、`pages`

### G3 HATEOAS
- 响应中包含 `_links` 字段，提供相关资源的链接
- 示例：论文详情中包含 `notes`、`save`、`self` 链接
- 列表响应包含 `next`、`prev`、`self` 分页链接

### G4 输入验证
- 所有用户输入进行验证（必填字段、类型、长度）
- 验证失败返回 400 并说明具体错误字段

### G5 CORS
- 配置 CORS 允许跨域请求（为 Swagger UI 和潜在前端扩展做准备）
